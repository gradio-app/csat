Exam Structure

Section I: Algorithms

The algorithms section consists of 4 questions over 2 hours worth 10 points each. In each of these questions, a function description and signature will be provided. Test cases to demonstrate function behaviour will also be provided. Candidates must implement each function with efficient time and space complexity, and also provide a complexity assessment of their own solution.

Each question will be graded with the following scoring rubric:

  • Valid, functional solution: 3 pts.
  • Correct assessment of complexities: 1 pt.
  • Use of optimal implementation: 6 pts.

The first 3 points are awarded on whether the candidate solution satisfies the function description. If it fails to do so, the remaning 7 points will not be considered.

The questions will cover the following topics:

  • Arrays
  • Stacks & Queues
  • Sets
  • Linked Lists
  • Hash Tables
  • Trees
  • Graphs
  • Sorting
  • Searching
  • Recursion & Dynamic Programming
  • Space & Time Complexities

Section II: Feature Implementation

This section will provide access to an existing codebase in an online editor along with documentation. The candidate will be asked to make changes to the codebase, including implementing new features, refactoring code, fixing bugs, and adding tests. The diff made on the codebase will be scored as follows:

  • Valid, functional implementation of requested features: 25 pts.
  • Optimal implementation: 5 pts.
  • Clean, highly readable code: 5 pts.
  • Test coverage for added features: 5 pts.

Clean code refers to code that is intuitive and easy to follow. Tips to score well here include: using clear variable and function names, dividing logic intuitively between functions, avoiding excessive nesting, avoiding hard coded values, following existing conventions used in the code, and sparse use of comments when necessary. Any new features should have appropriate test coverage, and any breaking tests after changes should be updated.
