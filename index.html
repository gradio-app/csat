CSAT Exam for Software Engineers.

The CSAT Exam assesses competence of individuals in modern software programming.

The exam focuses on algorithms, designs, and practices specifically relevant to software development used in industry today. CSAT aims to provide a transparent signal of engineering ability of individuals without bias towards background.

Exam Overview

Time: 4 hours
Score: 0 - 80 pts.
Section I: Algorithms
2 hours, 4 questions, worth 10 points each

In each question, a function description and signature are provided with test cases. Candidate must provide an efficient implementation.

Section II: Feature Implementation
2 hours, worth 40 points

Provided an existing codebase, the candidate must implement a requested set of features.

Assessment Goals of CSAT

CSAT aims to ensure a level field for hiring software talent by measuring the skills most relevant to modern software engineering. CSAT measures three engineering skills:

Working with Algorithms and Data Structures

CSAT measures a candidate's proficiency with implementing efficient algorithms. Given a function description and example test cases, the candidate is asked to write the function body and assess the space and time complexities of their own implementation.

Efficient solutions will require a strong understanding of data structures.

Reading Codebases and Documentation

In Section II of the exam, the candidate must familiarize themselves with an existing codebase by browsing through the code and available documentation.

Reading code is an essential skill for software engineers that translates strongly into productive engineering.

Writing Quality Code

The candidate will be graded on their diff to the codebase in Section II. In addition to implementing features, they may be required to refactor existing sections, clean bugs, and provide test cases.

The overall diff will be given points for completeness of implementation and quality of code written. CSAT rewards the ability of a candidate to maintain a healthy codebase.
